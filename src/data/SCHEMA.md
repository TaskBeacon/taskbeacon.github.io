# Task Index Schema

`src/data/tasks_index.json` is generated by `scripts/build-index.mjs`.

Schema version: `5`

## Top-level

- `schema_version`: number
- `generated_at`: ISO timestamp string
- `org`: string (GitHub org)
- `tasks`: array of task objects

## Task object

Required:

- `repo`: string (repo name)
- `full_name`: string (`Org/Repo`)
- `html_url`: string
- `default_branch`: string
- `short_description`: string
- `tags`: object
  - `paradigm`: string[]
  - `response`: string[]
  - `modality`: string[]
  - `language`: string[]
- `keywords`: string[]
- `last_updated`: ISO timestamp string
- `readme_snapshot_path`: string (public path like `/readmes/T000012-sst.md`)

Optional:

- `id`: string
- `slug`: string
- `title`: string
- `acquisition`: string | null
- `variant`: string | null
- `release_tag`: string | null
- `maturity`: string | null (for example `smoke_tested`, `piloted`)
- `psyflow_version`: string | null
- `has_voiceover`: boolean | null
- `structure`: object
  - `config`: boolean
  - `assets`: boolean
  - `src`: boolean
- `run_url`: string | null
- `readme_run_anchor`: string (for example `#run`)
- `web_variant`: object | null
  - `id`: string | null
  - `repo`: string
  - `title`: string
  - `html_url`: string
  - `default_branch`: string
  - `short_description`: string
  - `maturity`: string | null
  - `acquisition`: string | null
  - `variant`: string | null
  - `release_tag`: string | null
  - `last_updated`: ISO timestamp string
  - `run_url`: string
  - `download_zip`: string

## Notes

- `taskbeacon.yaml` and `taskbeacon.yml` are preferred metadata sources. The indexer still accepts `task.yaml`, `task.yml`, and `task.json` for backward compatibility.
- `short_description` is taken from metadata first, then README table fields, then the README first paragraph, then the GitHub repo description.
- `maturity` is taken from metadata when present; otherwise it is parsed from README badges like `![Maturity: smoke_tested]`.
- `readme_snapshot_path` points to a statically exported README snapshot under `public/readmes/`, so client-side UI can fetch markdown on demand without bundling README bodies into the initial page payload.
- `run_url` is used for standalone HTML tasks. By default it points to the shared `psyflow-web` GitHub Pages runner with `?task=<repo>`, unless metadata overrides it.
- `web_variant` is now runtime-attached by the website client after fetching `/psyflow-web/task-manifest.json`. In the generated `tasks_index.json`, canonical baseline tasks usually keep `web_variant: null`.
